# H-002 Handoff (Codex)

## Context
H-002 required deterministic sleeve-drift detection plus scheduler-facing automation so rebalance work can be surfaced as explicit intents per sleeve.

## Changes
1. `portfolio/rebalance.py`
   - Added deterministic drift planner types:
     - `RebalanceAction`
     - `RebalancePlan`
   - Added explicit rebalance intent layer:
     - `RebalanceIntent`
     - `build_rebalance_intents(plan)`
   - Added scheduler/pipeline wrapper:
     - `DriftPlanner`
   - `RebalancePlan.to_dict()` now includes:
     - `rebalance_intents`
     - `rebalance_intents_count`
   - Preserved configurable controls:
     - `drift_threshold_pct`
     - `min_trade_notional`
     - optional explicit target sleeve weights
2. `app/engine/scheduler.py`
   - Added optional `rebalance_check_fn` injection to `DailyWorkflowScheduler`.
   - Added post-dispatch best-effort hook `_run_rebalance_check(...)` with failure isolation.
3. `app/engine/pipeline.py`
   - Added `dispatch_rebalance_check(...)` callback that executes `run_rebalance_drift_check(...)` and returns scheduler-safe payloads.
4. Tests
   - Added new file `tests/test_rebalance.py` covering:
     - target-weight derivation
     - threshold behavior
     - stale/new sleeve handling
     - intent generation
     - planner wrapper behavior
   - Extended `tests/test_scheduler.py` for rebalance hook execution and failure-path behavior.
   - Extended `tests/test_pipeline.py` for `dispatch_rebalance_check(...)` success/error handling.
