# G-004 Handoff (Codex)

## Context
G-004 integrates G-003 AI panel consensus into execution-time gating, calibrated against G-002 live execution-quality outcomes. Design requirement preserved: kill-switch/router and risk hard-stops must run before AI gating, and exits must bypass AI gate.

## Changes
1. `app/signal/ai_confidence.py`
   - Added `ExecutionQualitySnapshot`, `AIConfidenceGateConfig`, `AIConfidenceDecision`.
   - Added calibrated confidence computation combining panel confidence/agreement/conviction and execution-quality modifiers.
   - Added gate decision helper with explicit reject reasons:
     - `ai_execution_quality_missing`
     - `ai_models_responded_below_min`
     - `ai_consensus_neutral`
     - `ai_confidence_below_threshold`
2. `execution/policy/ai_gate_policy.py`
   - Added policy wrapper (`AIGatePolicyInput`, `evaluate_ai_gate_policy`) for deterministic orchestrator integration.
3. `execution/policy/__init__.py`
   - Exported AI gate policy symbols.
4. `app/engine/orchestrator.py`
   - Added optional AI gate inputs:
     - `ai_consensus_by_ticker`
     - `ai_execution_quality`
     - `ai_gate_config`
   - Integrated gate after router+risk checks, entry-only.
   - Exits remain ungated by AI.
5. `app/engine/pipeline.py`
   - Added optional pass-through overrides for AI consensus/quality/config.
   - Added optional live panel collection path via `ai_panel_enabled=True`.
   - Added execution-quality snapshot builder for gate calibration.
6. Tests
   - Added `tests/test_ai_confidence.py`.
   - Extended `tests/test_orchestrator.py` with AI gate integration coverage.
   - Extended `tests/test_pipeline.py` with AI pass-through and enabled-path wiring coverage.
