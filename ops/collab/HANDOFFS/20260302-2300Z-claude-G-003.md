## Risks
- LLM response parsing: models may not always return clean JSON. Three fallback strategies in `_parse_json_from_response()` (direct, code fence, brace-finding). Parse failures recorded as model failures in PanelConsensus.
- API rate limits: each client has independent retry with exponential backoff. Sequential model querying avoids aggregate rate limit spikes.
- No live API validation yet — clients tested via FakeSession pattern. Real API keys needed for live verification.
