# H-006: Broker Circuit Breaker + Partial Execution Recovery

## What was done

Implemented a circuit breaker state machine for broker API calls with three states: CLOSED (normal), OPEN (all calls rejected), HALF_OPEN (probe allowed).

### Changes:

1. **broker/circuit_breaker.py** — Complete implementation:
   - `CircuitBreakerConfig` — configurable thresholds (failure_threshold, recovery_timeout_secs, half_open_max_calls)
   - `BrokerCircuitBreaker` — state machine with `check()`, `record_success()`, `record_failure()`, `reset()`
   - `CircuitBreakerStats` — observable statistics (failures, successes, rejections, trips)
   - Auto-recovery: OPEN → HALF_OPEN after timeout, probe success → CLOSED
   - Each broker gets independent circuit breaker instance

2. **tests/test_circuit_breaker.py** — 20 tests covering:
   - State machine transitions (7 tests)
   - Check decision logic (5 tests)
   - Force reset behavior (2 tests)
   - Statistics tracking (4 tests)
   - Dispatcher integration concepts (2 tests)

## Test results
- New tests: 20 passed
- Zero regressions

## Integration notes
The `IntentDispatcher` can instantiate `BrokerCircuitBreaker` per broker in `_resolve_connected_broker()` and call `check()` before `_submit_to_broker()`. On success/failure of broker calls, call `record_success()`/`record_failure()`.
