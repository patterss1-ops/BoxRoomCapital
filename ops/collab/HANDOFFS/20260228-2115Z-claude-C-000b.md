## Changes

### Fix 1: Sleeve query truncation (`data/trade_db.py`)
- `get_sleeve_daily_reports()` `days` parameter now selects the N most recent *distinct dates*, returning all sleeve rows for those dates.
- Previously used global `LIMIT N` which returned only N total rows — silently dropping sleeves when >N sleeves existed on a single date.
- Implementation: subquery `SELECT DISTINCT report_date ... LIMIT ?` used in `WHERE report_date IN (...)` clause.

### Fix 2: day_pnl derivation (`risk/portfolio_risk.py`)
- `get_risk_briefing()` now derives day_pnl correctly from previous NAV.
- Previous formula: `day_pnl = total_nav * daily_return_pct / 100` (uses current NAV as base — incorrect).
- New formula: `prev_nav = total_nav / (1 + r/100)`, `day_pnl = total_nav - prev_nav` (mathematically consistent with return definition).
- For 2% return on 100k NAV: old = 2000.00, new = 1960.78 (correct).
