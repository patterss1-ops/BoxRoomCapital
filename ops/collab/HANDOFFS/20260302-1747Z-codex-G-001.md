# G-001 Handoff (Codex)

## Context
Phase G starts with execution telemetry so AI confidence can be calibrated on real execution outcomes (fills/rejects/slippage), not backtests.

## Changes
1. `data/order_intent_store.py`
   - Extended schema with `order_execution_metrics` table + indexes.
   - Added `record_execution_metric(...)` for per-attempt upserted telemetry rows.
   - Added `get_execution_metrics(...)` query helper for downstream reporting.
   - Added slippage/latency helpers:
     - reference-price extraction from intent/response payloads
     - side-aware slippage in bps
     - dispatch-to-broker timestamp latency (ms)
2. `execution/dispatcher.py`
   - Wired telemetry persistence for completed/retrying/failed outcomes.
   - Records metrics for broker rejects and dispatch exceptions.
   - Preserves existing lifecycle transitions and retry semantics.
3. `tests/test_dispatcher.py`
   - Added coverage for completed-fill telemetry (including slippage/notional fields).
   - Added coverage for retrying-reject telemetry (error metadata path).

## Tests/Checks

- `python3 -m pytest -q tests` -> 932 passed

## Risks
- Slippage depends on optional reference-price metadata; missing reference data yields `slippage_bps=NULL` by design.
- Timestamp quality depends on broker adapters returning `OrderResult.timestamp` consistently.
