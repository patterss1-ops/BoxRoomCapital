# D-002 Handoff (Codex)

## Context
Phase D execution loop gap: intents were persisted as `queued`/`retrying` but nothing consumed them for broker submission.

## Changes
- Added `execution/dispatcher.py` with `IntentDispatcher`:
  - polls dispatchable intents (queued/retrying)
  - transitions `queued/retrying -> running -> completed/retrying/failed`
  - submits to broker adapters via resolver
  - supports entry and exit dispatch (`metadata.is_exit` routes to `close_position`)
  - writes structured response/error payloads into order-intent lifecycle records
- Extended `data/order_intent_store.py`:
  - new `get_dispatchable_order_intents(limit, db_path)` helper
  - joins `order_intents` + `order_actions`
  - filters to intents with remaining attempt budget
  - returns oldest-first dispatch order
- Added `tests/test_dispatcher.py` covering:
  - successful queued execution
  - retry then complete path
  - terminal failure on exhausted attempts
  - exit intent close path
  - unsupported broker path -> retrying
  - dispatchable query ordering
