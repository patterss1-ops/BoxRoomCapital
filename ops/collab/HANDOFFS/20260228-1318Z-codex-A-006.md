## Context
Ticket `A-006` adds a hard pre-trade risk gate that blocks unsafe submissions before broker calls and emits machine-readable risk decision records.

## Changes
- Added `risk/pre_trade_gate.py`:
  - `RiskLimits` (`max_position_pct_equity`, `max_sleeve_pct_equity`, `max_correlated_pct_equity`)
  - `RiskOrderRequest`
  - `RiskContext` (kill switch state, cooldowns, exposure maps)
  - `RiskDecision` with `rule_id`, observed/threshold values, and `to_audit_payload()`
  - `evaluate_pre_trade_risk(...)` hard-rule evaluation order:
    1. `KILL_SWITCH_ACTIVE`
    2. `MARKET_COOLDOWN_ACTIVE`
    3. `MAX_POSITION_PCT_EQUITY`
    4. `MAX_SLEEVE_PCT_EQUITY`
    5. `MAX_CORRELATED_PCT_EQUITY`
    6. `APPROVED`
- Added `execution/risk_gate.py`:
  - `enforce_pre_trade_risk(...)` (raises `RiskGateRejectedError` on reject)
  - `submit_with_risk_gate(...)` wrapper that guarantees no submit call when rejected
  - `RiskGateResult` envelope for caller integration
- Added `risk/__init__.py` exports.
- Added `tests/test_risk_gate.py`:
  - per-rule reject coverage
  - explicit `rule_id` and threshold assertions
  - enforcement exception behavior
  - broker-call blocking verification on rejection
  - positive-path submit execution verification

## Risks
- This ticket introduces the gate and execution wrapper but does not yet wire every production execution path to call it; integration into live/manual submission entry points remains pending.
- Correlated exposure currently relies on caller-provided `correlated_group` and exposure maps; portfolio-level factor model integration is a follow-up step.
