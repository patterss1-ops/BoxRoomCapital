## Changes
- Extended `data/trade_db.py` with 5 new multi-broker ledger tables:
  - `broker_accounts`: Broker account registry with broker, account_id, account_type (ISA/SIPP/GIA/SPREADBET/PAPER), currency, label, active flag
  - `broker_positions`: Position tracking per broker account with ticker, direction, quantity, avg_cost, market_value, unrealised_pnl, strategy/sleeve attribution, con_id
  - `broker_cash_balances`: Time-series cash balance snapshots per broker account
  - `nav_snapshots`: NAV snapshots at fund/sleeve/account level with net_liquidation, cash, positions_value, unrealised/realised P&L
  - `reconciliation_reports`: Reconciliation audit trail with status, positions_checked, mismatches_found, JSON details

- Created `execution/ledger.py` (~418 lines):
  - `register_broker_account()`: Register broker accounts with UUID, ON CONFLICT upsert
  - `get_broker_accounts()`: List accounts with optional broker/active filters
  - `sync_positions()`: Full position sync from broker — inserts new, updates existing, removes stale positions. Returns sync summary {synced, inserted, updated, removed}
  - `get_unified_positions()`: Cross-broker unified position view with JOIN to broker_accounts. Filter by broker or sleeve
  - `sync_cash_balance()`: Record cash balance snapshot for a broker account
  - `get_latest_cash_balances()`: Most recent cash balance per active broker account using MAX(synced_at) subquery
  - `save_nav_snapshot()`: Persist NAV at fund/sleeve/account level with ON CONFLICT upsert by (date, level, level_id)
  - `get_nav_history()`: NAV time series for charting, ordered DESC with LIMIT
  - `reconcile_positions()`: Compare broker-reported positions vs ledger positions. Detects: quantity_mismatch, missing_in_ledger, phantom_in_ledger. Each mismatch includes ticker, direction, both quantities, and corrective suggestion text. Persists reconciliation report to DB
  - `get_reconciliation_reports()`: Historical reconciliation reports with JSON details parsing

- Created `tests/test_ledger.py` (~450 lines, 41 tests):
  - TestBrokerAccountRegistration: 7 tests — register, conflict upsert, filters, inactive exclusion, field persistence
  - TestPositionSync: 6 tests — insert, update, remove stale, empty sync, optional fields, same ticker different directions
  - TestUnifiedPositions: 4 tests — all brokers, filter by broker, filter by sleeve, account metadata enrichment
  - TestCashBalance: 4 tests — sync, latest-only retrieval, multi-broker, account metadata
  - TestNAVSnapshots: 7 tests — fund/sleeve/account level, upsert same date, multiple dates, ordering, realised P&L
  - TestReconciliation: 11 tests — clean, qty mismatch, missing in ledger, phantom in ledger, multiple mismatches, report persistence, JSON details, empty broker, limit, all accounts, suggestion text
  - TestLedgerIntegration: 2 tests — full multi-broker lifecycle, position update then reconcile
