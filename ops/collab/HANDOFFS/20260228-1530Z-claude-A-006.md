## 12-Rule Risk Gate (R-001 through R-012)

| Rule | Name | Level | Description |
|------|------|-------|-------------|
| R-001 | kill_switch | Fund | Block all trades when kill switch active |
| R-002 | fund_drawdown | Fund | Block if fund drawdown exceeds 30% |
| R-003 | daily_loss_limit | Fund | Block if daily loss exceeds 5% |
| R-004 | cash_buffer | Fund | Maintain 10% NAV in cash minimum |
| R-005 | trade_risk_limit | Trade | Max 2% risk per trade |
| R-006 | trade_notional_limit | Trade | Max 10% notional per trade |
| R-007 | position_concentration | Position | No single ticker > 5% of NAV |
| R-008 | sleeve_allocation | Sleeve | No sleeve > 35% of fund NAV |
| R-009 | sleeve_drawdown | Sleeve | Block trades in sleeve with > 15% DD |
| R-010 | strategy_max_positions | Strategy | Max open positions per strategy |
| R-011 | sector_concentration | Portfolio | No sector > 25% of NAV |
| R-012 | cooldown | Timing | 4h cooldown after kill switch reset |

## Architecture
- `OrderProposal`: Broker-agnostic trade proposal (ticker, direction, qty, notional, risk, strategy, sleeve, sector)
- `PortfolioSnapshot`: Current portfolio state (fund NAV, cash, peak, positions, sleeve NAVs, kill switch state)
- `RiskVerdict`: Machine-readable result with rule_id, reason, threshold, actual value, checks_run count
- `build_portfolio_snapshot()`: Builds PortfolioSnapshot from A-005 ledger data
- All limits configurable via frozen `RiskLimits` dataclass
