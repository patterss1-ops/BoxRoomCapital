# E-007 Handoff (Codex)

## Context
Ticket E-007 implements a shadow-only Signal Engine runtime surface on top of E-006 composite scoring so operators can run and inspect scoring cycles without dispatching live orders.

## Changes
- Added `app/engine/signal_shadow.py`:
  - collects latest `signal_layer` events from `research_events`
  - computes per-ticker composite decisions via `evaluate_composite`
  - persists latest report to `strategy_state` (`signal_shadow:last_report`)
  - exposes snapshot helper with coverage stats for operator/API surfaces
- Updated `app/api/server.py`:
  - `GET /api/signal-shadow`
  - `POST /api/actions/signal-shadow-run`
  - `GET /fragments/signal-engine`
  - async job runner `_run_signal_shadow_job` with `jobs` lifecycle persistence
- Added `app/web/templates/_signal_engine.html` operator panel:
  - run button for shadow cycle
  - latest run summary table (status/score/action/layer coverage)
  - live layer event coverage table
- Added `tests/test_signal_shadow_api.py`:
  - API snapshot route
  - action/job lifecycle
  - fragment render
  - integration test for latest-layer selection + persisted snapshot

## Risks
- Shadow cycle currently uses whatever layer events exist and marks tickers with too few layers as `insufficient_layers`; production calibration will likely tune `min_layers_for_score` and freshness thresholds.
- Operator panel is delivered as fragment/API surface; embedding into a top-level page can be done in follow-up UX refinement without changing runtime semantics.
- Regression impact: touches API routing in `server.py`; mitigated by dedicated endpoint/fragment tests and full-suite green.
- Rollback note: revert this ticket commit(s) to remove shadow endpoints/module/template/tests cleanly.
