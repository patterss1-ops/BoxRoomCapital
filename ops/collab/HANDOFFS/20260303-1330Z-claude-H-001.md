# H-001: Shadow‚ÜíStaged‚ÜíLive Promotion Enforcement

## What was done

Wired the existing `fund/promotion_gate.py` reporting module into the execution pipeline as a hard enforcement gate. Entries are now blocked unless the strategy has completed the full promotion pipeline.

### Changes:

1. **fund/promotion_gate.py** ‚Äî Added enforcement layer:
   - `PromotionGateConfig` dataclass (enabled, min_soak_hours, max_stale_hours, require_live_set, bypass_for_exits)
   - `PromotionGateDecision` dataclass (allowed, reason_code, message, metadata)
   - `evaluate_promotion_gate()` ‚Äî deterministic enforcement function

2. **app/engine/orchestrator.py** ‚Äî Integration:
   - Added `promotion_gate_config` parameter to `run_orchestration_cycle()`
   - Added `promotion_gate_config` to `_process_one_signal()`
   - Enforcement runs after risk gate, before AI confidence gate
   - Entry-only: exits bypass via config flag

3. **tests/test_promotion_enforcement.py** ‚Äî 14 tests covering:
   - No live set blocks entry
   - Live set allows entry (after soak)
   - Exit bypass behavior (on/off)
   - Gate disabled passthrough
   - Soak period enforcement (blocks during window, passes after)
   - Stale set detection
   - Live not required config
   - Decision metadata inclusion
   - Zero soak hours edge case
   - Orchestrator integration (3 tests): rejects without live set, allows with valid set, no gate when config is None

### Enforcement rules:
1. **Require live set** ‚Äî Strategy must have an active `live` lane parameter set
2. **Soak period** ‚Äî Configurable minimum hours since last promotion to live (default 24h)
3. **Stale detection** ‚Äî Configurable maximum hours since last promotion (default 168h/7d)
4. **Exit bypass** ‚Äî Exits always pass when enabled (default)

## Test results
- New tests: 14 passed
- Existing promotion_gate tests: 14 passed (zero regressions)
- Existing orchestrator tests: 36 passed (zero regressions)

## Regression impact
- Backward compatible: `promotion_gate_config=None` (default) means gate is not active
- No existing behavior changes unless explicitly enabled

## Rollback
Revert fund/promotion_gate.py enforcement additions and orchestrator wiring.
