# F-004 Handoff — L6 News Sentiment Scorer + Feed Normalizer

## Context
Phase F contract freeze (`F-001`) is merged. `F-004` delivers the L6 implementation lane owned by Codex.

## Changes
- Added `app/signal/layers/news_sentiment.py`:
  - `NewsArticle` normalized input contract
  - deterministic L6 scorer with:
    - source-quality weighting
    - recency weighting over configurable window
    - negativity-bias penalty for clustered negative coverage
  - outputs canonical `LayerScore` (`LayerId.L6_NEWS_SENTIMENT`)
  - guarantees required detail keys from `F-001` contract:
    - `sentiment_polarity`
    - `article_count`
    - `negative_article_ratio`
    - `window_hours`
  - deterministic `provenance_ref` generation.
- Added `intelligence/news_sentiment.py`:
  - feed normalization helpers across mixed provider schemas
  - label/numeric/% sentiment normalization into `[-1, 1]`
  - epoch/ISO timestamp normalization
  - ticker-grouped batch scoring helpers.
- Added `tests/test_signal_layer_news_sentiment.py`:
  - normalization behavior tests
  - scoring regime tests (positive, neutral, negative spike, stale/out-of-window)
  - required-key contract tests
  - deterministic provenance tests
  - batch feed scoring tests.
